---
title: "Substring Analysis"
description: "How Textee extracts and counts 1-, 2-, and 3-word substrings from text."
---

`ParseString` is the core of Textee. It splits input text into sentences, tokenizes each sentence into words, generates n-grams of width 1, 2, and 3, cleans each n-gram, and stores occurrence counts in an atomic-safe map. This page walks through every step of that pipeline.

## Step 1 â€” Sentence splitting

`ParseString` calls `stringToSentenceSlice`, which applies this compiled regex to the input:

```go
var regFindSentences = regexp.MustCompile(`(?m)([^.!?]*[.!?])(?:\s|$)`)
```

The regex matches any run of non-terminal characters followed by `.`, `!`, or `?`, and then either whitespace or end-of-string. Every match becomes one sentence.

<Note>
If the regex finds no matches â€” meaning the input contains no `.`, `!`, or `?` â€” the entire input string is returned as a single-element slice. The text is never silently dropped.
</Note>

```go
func stringToSentenceSlice(text string) ([]string, error) {
    if regFindSentences == nil {
        return nil, ErrRegexpMissing
    }
    matches := regFindSentences.FindAllString(text, -1)
    for i, match := range matches {
        matches[i] = strings.TrimSpace(match)
    }
    if len(matches) == 0 {
        return []string{text}, nil
    }
    return matches, nil
}
```

## Step 2 â€” Word tokenization

For each sentence, `strings.Fields` splits on any run of whitespace. This handles multiple spaces, tabs, and mixed spacing without any extra configuration.

```go
words := strings.Fields(sentence)
```

`strings.Fields("Hello   world\tfoo")` returns `["Hello", "world", "foo"]`.

## Step 3 â€” Sliding window n-gram generation

A nested loop generates all 1-, 2-, and 3-word n-grams starting at every word position:

```go
for i := 0; i < len(words); i++ {
    for j := i + 1; j <= i+3 && j <= len(words); j++ {
        substring := strings.Join(words[i:j], " ")
        // ...
    }
}
```

The inner bound `j <= i+3` caps the window at three words. The slice `words[i:j]` is joined with a single space to form the raw n-gram string.

## Step 4 â€” Cleaning each n-gram

Two transformations are applied to every raw n-gram before storage:

1. **Strip non-alphanumeric characters** â€” the regex `[^a-zA-Z0-9\s]` removes all punctuation, apostrophes, hyphens, and special characters.
2. **Lowercase and trim** â€” `strings.ToLower` and `strings.TrimSpace` normalize the result.

```go
var regCleanSubstring = regexp.MustCompile(`[^a-zA-Z0-9\s]`)

func cleanSubstring(word string) (string, error) {
    if regCleanSubstring == nil {
        return "", ErrRegexpMissing
    }
    word = strings.TrimSpace(word)
    return regCleanSubstring.ReplaceAllString(word, ""), nil
}
```

After calling `cleanSubstring`, `ParseString` applies the remaining two steps:

```go
cleanedSubstring = strings.ToLower(cleanedSubstring)
cleanedSubstring = strings.TrimSpace(cleanedSubstring)
```

Empty strings that result from cleaning (e.g., a token made entirely of punctuation) are skipped:

```go
if cleanedSubstring != "" {
    // store in map
}
```

## Step 5 â€” Atomic occurrence counting

The `Substrings` field is `map[string]*atomic.Int32`. On first encounter a new `*atomic.Int32` is allocated; on every subsequent encounter `Add(1)` increments the counter. A mutex guards map reads and writes while the per-value increment is lock-free:

```go
tt.mu.Lock()
if _, ok := tt.Substrings[cleanedSubstring]; !ok {
    tt.Substrings[cleanedSubstring] = new(atomic.Int32)
}
tt.Substrings[cleanedSubstring].Add(1)
tt.mu.Unlock()
```

## Accessing the raw `Substrings` map

After calling `NewTextee` (or `ParseString` directly), the `Substrings` field is safe to read â€” provided no concurrent mutation is happening. Call `.Load()` on each value to retrieve the current count:

```go
tt, err := textee.NewTextee("Hello world foo bar")
if err != nil {
    log.Fatal(err)
}

for substring, counter := range tt.Substrings {
    fmt.Printf("%q: %d\n", substring, counter.Load())
}
```

<Tip>
Map iteration order in Go is randomized. If you need a stable ordering â€” e.g., most-frequent first â€” use `tt.SortedSubstrings()` instead. See the [Sorting Results](/guides/sorting-results) guide for details.
</Tip>

## Error handling

`ParseString` collects `CleanError` values from the cleaning step and joins them with `errors.Join` before returning. This means a single bad token does not abort processing â€” other substrings are still counted. Check the returned error before using the result:

```go
tt, err := textee.NewTextee(input)
if err != nil {
    // errors.Is / errors.As work with the joined error tree
    log.Fatal(err)
}
```
